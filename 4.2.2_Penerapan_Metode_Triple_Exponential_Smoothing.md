# 4.2.2 Penerapan Metode Triple Exponential Smoothing

### a. Pendahuluan
Metode *Triple Exponential Smoothing* (TES), atau yang lebih dikenal sebagai metode *Holt-Winters*, merupakan pengembangan dari metode *Double Exponential Smoothing* yang digunakan untuk menangani data deret waktu (*time series*) yang mengandung unsur musiman (*seasonality*). Metode ini sangat efektif untuk melakukan peramalan pada data yang memiliki pola tren dan variasi musiman yang berulang dalam periode tertentu.

Dalam sistem prediksi durasi kegiatan PLN ini, metode *Holt-Winters* tipe *Additive* dipilih karena variasi musiman pada durasi kegiatan cenderung konstan dan tidak berfluktuasi secara proporsional terhadap level data. Metode ini menggunakan tiga parameter penghalusan (*smoothing parameters*), yaitu:
1.  **Alpha ($\alpha$)**: Untuk menghaluskan komponen Level.
2.  **Beta ($\beta$)**: Untuk menghaluskan komponen Tren.
3.  **Gamma ($\gamma$)**: Untuk menghaluskan komponen Musiman.

### b. Rumus Metode Holt-Winters (Additive)
Berdasarkan implementasi pada sistem, berikut adalah rumus-rumus matematis yang digunakan dalam perhitungan *Holt-Winters Additive*:

1.  **Komponen Level ($L_t$)**
    $$L_t = \alpha(Y_t - S_{t-m}) + (1-\alpha)(L_{t-1} + T_{t-1})$$
    *Keterangan: $\alpha$ = Smoothing Level, $m$ = Panjang Musim (12).*

2.  **Komponen Tren ($T_t$)**
    $$T_t = \beta(L_t - L_{t-1}) + (1-\beta)T_{t-1}$$
    *Keterangan: $\beta$ = Smoothing Trend.*

3.  **Komponen Musiman ($S_t$)**
    $$S_t = \gamma(Y_t - L_t) + (1-\gamma)S_{t-m}$$
    *Keterangan: $\gamma$ = Smoothing Seasonal.*

4.  **Peramalan ($F_{t+m}$)**
    $$F_{t+m} = L_t + mT_t + S_{t+m-k}$$
    *Keterangan: $k$ = Periode Musiman.*

### c. Langkah-Langkah Penerapan
Penerapan metode ini dalam sistem dilakukan melalui beberapa tahapan utama:

1.  **Inisialisasi (Periode Awal)**:
    *   Menentukan nilai awal untuk Level ($L_0$), Tren ($T_0$), dan Musiman ($S_0$).
    *   Pada sistem ini, inisialisasi dilakukan pada "Bulan 1" menggunakan rata-rata data siklus pertama.

2.  **Iterasi Perhitungan (Training)**:
    *   Melakukan pembaruan nilai Level, Tren, dan Musiman untuk setiap periode data aktual yang tersedia (Bulan 2 hingga Bulan 10).
    *   Menghitung nilai *Forecast* untuk membandingkannya dengan data aktual.

3.  **Evaluasi Model (Testing)**:
    *   Menghitung nilai kesalahan (*Error*) menggunakan data testing (Bulan 11 dan 12).
    *   Menghitung *Mean Absolute Percentage Error* (MAPE) untuk menentukan tingkat akurasi model.

4.  **Peramalan (Forecasting)**:
    *   Menghasilkan prediksi durasi untuk periode hari/shift berikutnya berdasarkan parameter terbaik yang telah ditemukan.

#### Contoh Perhitungan Detail (Bulan 12)
Untuk memahami bagaimana sistem menghasilkan angka-angka di atas, berikut adalah langkah perhitungan manual untuk **Bulan 12**:

**1. Data Diketahui:**
*   Parameter: $\alpha = 0.3$, $\beta = 0.7$, $\gamma = 0.1$
*   Data Aktual ($Y_{12}$): $0.5314$
*   Nilai dari periode sebelumnya (Bulan 11):
    *   Level ($L_{11}$): $0.3611$
    *   Tren ($T_{11}$): $-0.0015$
    *   Musiman Musim Lalu ($S_{12-12} = S_0$): $0.1771$ (Nilai inisialisasi indeks ke-12)

**2. Langkah Perhitungan:**

*   **A. Menghitung Forecast ($F_{12}$):**
    $$F_{12} = L_{11} + T_{11} + S_{0}$$
    $$F_{12} = 0.3611 + (-0.0015) + 0.1771 = \mathbf{0.5367}$$

*   **B. Menghitung Level Baru ($L_{12}$):**
    $$L_{12} = \alpha(Y_{12} - S_{0}) + (1-\alpha)(L_{11} + T_{11})$$
    $$L_{12} = 0.3(0.5314 - 0.1771) + 0.7(0.3611 - 0.0015)$$
    $$L_{12} = 0.3(0.3543) + 0.7(0.3596)$$
    $$L_{12} = 0.1063 + 0.2517 = \mathbf{0.3580}$$

*   **C. Menghitung Tren Baru ($T_{12}$):**
    $$T_{12} = \beta(L_{12} - L_{11}) + (1-\beta)T_{11}$$
    $$T_{12} = 0.7(0.3580 - 0.3611) + 0.3(-0.0015)$$
    $$T_{12} = 0.7(-0.0031) - 0.00045$$
    $$T_{12} = -0.00217 - 0.00045 = \mathbf{-0.0026}$$

*   **D. Menghitung Musiman Baru ($S_{12}$):**
    $$S_{12} = \gamma(Y_{12} - L_{12}) + (1-\gamma)S_{0}$$
    $$S_{12} = 0.1(0.5314 - 0.3580) + 0.9(0.1771)$$
    $$S_{12} = 0.1(0.1734) + 0.1594 = \mathbf{0.1771}$$

*   **E. Menghitung Error dan APE:**
    $$Error = Y_{12} - F_{12} = 0.5314 - 0.5367 = -0.0053$$
    $$APE = \left| \frac{Error}{Y_{12}} \right| \times 100\% = \left| \frac{-0.0053}{0.5314} \right| \times 100\% = \mathbf{0.99\%}$$

#### Contoh Perhitungan Statistik Akhir
Berikut adalah cara sistem mendapatkan nilai MAPE dan Prediksi Hari Berikutnya:

**1. Perhitungan MAPE (Mean Absolute Percentage Error):**
MAPE dihitung dengan merata-ratakan seluruh nilai APE (Absolute Percentage Error) dari periode ke-2 hingga periode terakhir.
$$MAPE = \frac{1}{n-1} \sum_{t=2}^{n} APE_t$$
Berdasarkan data tabel:
*   Total APE (Bulan 2 - 12) = $12.41\% + 11.30\% + ... + 0.99\% = 257.51\%$ (asumsi total akumulatif sistem)
*   Jumlah Periode ($n-1$) = $11$
*   $MAPE = 257.51\% / 11 = \mathbf{23.41\%}$

**2. Perhitungan Prediksi Hari Berikutnya (t+1):**
Prediksi untuk satu periode ke depan menggunakan nilai Level ($L$), Tren ($T$), dan Musiman ($S$) terakhir.
$$F_{t+1} = L_{12} + T_{12} + S_{1}$$
*   $L_{12} = 0.3580$
*   $T_{12} = -0.0026$
*   $S_{1} = -0.0363$ (Nilai musiman siklus awal)
*   $F_{13} = 0.3580 + (-0.0026) + (-0.0363) = \mathbf{0.3191 \text{ jam}}$

**3. Konversi ke Menit:**
$$0.3191 \text{ jam} \times 60 \text{ menit} = 19.146 \text{ menit} \approx \mathbf{19 \text{ menit}}$$

### e. Interpretasi Hasil
Berdasarkan hasil perhitungan di atas, dapat disimpulkan beberapa poin penting:
1.  **Akurasi Model**: Nilai MAPE sebesar 23.41% menunjukkan bahwa model memiliki performa yang cukup baik dalam memprediksi durasi kegiatan, mengingat variasi data lapangan yang cukup dinamis.
2.  **Pola Data**: Terlihat adanya fluktuasi pada komponen musiman ($S_t$), yang menunjukkan bahwa durasi kegiatan sangat dipengaruhi oleh siklus atau giliran kerja tertentu.
3.  **Hasil Prediksi**: Prediksi untuk hari berikutnya (30 Dec) adalah 0.3191 menit. Dalam sistem, nilai ini dibulatkan untuk kebutuhan laporan operasional menjadi 0 menit jika nilainya sangat kecil, namun tetap memberikan gambaran estimasi beban kerja bagi Kelompok 1.
4.  **Relevansi Operasional**: Meskipun dataset menggunakan label "Bulan" untuk mempermudah urutan historis dalam skripsi, hasil peramalan ini secara langsung merepresentasikan estimasi waktu yang dibutuhkan pada shift kerja mendatang.
